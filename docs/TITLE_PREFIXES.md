# Title Prefixes

Use one of these bracketed prefixes at the start of your issue or pull request title. This helps our team understand the scope of the change at a glance.

## Core Services

- `[CORE]` - Shared engine logic, domain models, and workspace primitives.
- `[CLI]` - Command-line interface, argument parsing, and shell interactions.
- `[AUTH]` - Authentication, provider credentials, and account settings.
- `[STORAGE]` - SQLite persistence, state management, and indexing.
- `[OBSERVABILITY]` - Logging, diagnostics, telemetry, and error reporting.

## Agent & Intelligence

- `[AGENT]` - Orchestration, task planning, and agent lifecycle.
- `[MODEL]` - LLM provider routing, fallback policies, and response normalization.
- `[PROMPT]` - Prompt templates, system instructions, and authoring.
- `[SKILLS]` - Skill registration and execution metadata.
- `[TOOLS]` - Tool execution (web search, writer, file, etc.).
- `[CONTEXT]` - Prompt assembly and memory management.

## Developer & System

- `[ACP]` - Agent Communication Protocol support.
- `[MCP]` - Model Context Protocol support.
- `[LSP]` - Language Server Protocol integration.
- `[GIT]` - Git-aware features and repository context.
- `[AUTOMATION]` - Lifecycle hooks, triggers, and automated workflows.
- `[PACKAGING]` - Build pipelines, release strategy, and installers.
- `[SECURITY]` - Permissions, secrets, and trust boundaries.

## Frontend & UX

- `[UI]` - Desktop interface, navigation, and window management.
- `[CHAT]` - Conversation panels and interaction design.
- `[SETTINGS]` - Visual configuration screens.

## Maintenance & Policy

- `[BUG]` - Correcting broken or incorrect behavior.
- `[DOCS]` - Documentation-only changes.
- `[TEST]` - Test infrastructure and coverage.
- `[REFACTOR]` - Structural changes with no behavior change.
- `[CHORE]` - Maintenance and housekeeping.
